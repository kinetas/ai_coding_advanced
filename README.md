# AI Coding Advanced

AI와 AI 코딩 환경 관련 핵심 개념 정리.  

## 1. AI와 연산 장치

### NPU란?
`NPU(Neural Processing Unit)`는 AI 전용 연산을 빠르게 처리하려고 만든 반도체

- CPU: 범용 연산에 강함
- GPU: 병렬 연산에 강해서 AI 학습/추론에 많이 씀
- NPU: AI 연산에 특화되어 있어서 전력 효율, 속도 면에서 유리함

핵심은 NPU가 AI 모델이 자주 하는 행렬 연산, 추론 연산을 더 효율적으로 처리하는 전용 장치

## 2. AI 모델의 기본 특성

AI 모델은 대화를 잘하는 것처럼 보이지만, 본질적으로는 `무상태(stateless)`에 가까움.

- 한 번의 요청이 끝나면 그 자체로는 이전 내용을 계속 기억하지 못함
- 대화가 이어지는 것처럼 보이는 이유는 이전 대화 내용을 다시 같이 보내기 때문
- 그래서 "AI가 스스로 기억한다"기보다 시스템이 대화 이력을 관리한다고 보는 게 더 정확함

### 단기 기억처럼 보이는 이유
사용자는 AI가 이전 맥락을 기억한다고 느끼지만, 실제 흐름은 대체로 아래와 같음.

`사용자 <-> 챗봇 시스템 <-> AI API`

이 과정에서 챗봇 시스템이 이전 대화 내용을 저장해두고, 다음 요청 때 다시 AI에게 보내면 연속적인 대화처럼 보이게 함.

## 3. AI의 기억은 어떻게 유지되는가

AI 자체는 일회성 요청만 처리하는 경우가 많아서, 이전 내용을 기억하게 하려면 별도 저장 장치가 필요함.

일반적인 구조는 다음과 같음.

1. 사용자가 질문을 보냄
2. 챗봇 서비스가 이전 대화 기록을 DB에서 가져옴
3. 이전 대화와 현재 질문을 함께 AI 모델에 전달함
4. AI가 맥락을 반영해 답변함
5. 새 대화 내용을 다시 DB에 저장함

즉, `기억`은 AI 모델 내부가 아니라 보통 `DB`나 별도 저장소에서 관리.

### 기억이 사라질 수 있는 이유

기억이 유지되지 않는 이유는 여러 가지

- 저장 공간 부족으로 오래된 기록을 삭제하는 경우
- 기술적 결함이나 저장 실패가 발생한 경우
- 토큰 제한 때문에 오래된 대화를 일부 잘라내는 경우
- 세션이 새로 시작되어 이전 기록을 불러오지 못한 경우

## 4. AI 모델의 표준 대화 형식

대부분의 LLM API는 `messages` 리스트 형태로 대화를 주고받음.

각 메시지는 보통 아래 두 가지 핵심 요소로 구성

- `role`: 누가 말했는지 나타내는 역할
- `content`: 실제 내용

### 대표적인 role

| role | 의미 |
| --- | --- |
| `system` | AI의 행동 원칙, 말투, 제약 조건 설정 |
| `user` | 사용자의 질문이나 명령 |
| `assistant` | AI의 응답 |

### 예시

```json
[
  {
    "role": "system",
    "content": "당신은 사용자에게 명확하고 간결하게 답변하는 AI 비서입니다."
  },
  {
    "role": "user",
    "content": "오늘 날씨는 어때?"
  },
  {
    "role": "assistant",
    "content": "오늘은 맑고 화창한 날씨입니다."
  }
]
```

## 5. 왜 프롬프트를 잘 작성해야 하는가

프롬프트는 단순한 질문이 아니라, AI가 어떤 방식으로 답해야 하는지를 간접적으로 제어하는 수단

좋은 프롬프트를 작성하면 장점

- 원하는 형식으로 답을 받을 가능성이 높아짐
- 불필요한 추측이나 엉뚱한 답변이 줄어듦
- 시스템 메시지의 의도를 더 잘 반영하게 함
- 복잡한 작업을 단계적으로 시킬 수 있음

즉, 프롬프트를 잘 작성하는 이유는 AI 내부 시스템을 직접 건드릴 수 없더라도 입력을 통해 원하는 결과를 어느 정도 유도할 수 있기 때문.

## 6. 토큰이란?

토큰은 AI가 문장을 처리할 때 나누는 최소 단위.  
보통 단어랑 비슷하게 생각해도 되지만, 실제로는 단어보다 더 잘게 쪼개질 수도 있음.

즉, 문장 하나도 여러 개의 토큰으로 나뉘어서 처리

### 토큰이 중요한 이유

- 입력이 길수록 토큰 사용량이 증가함
- 출력이 길수록 더 많은 토큰이 필요함
- 대부분의 AI 모델은 한 번에 처리할 수 있는 최대 토큰 수가 정해져 있음
- 토큰 사용량은 비용과도 직접 연결되는 경우가 많음

그래서 실무에서는 대화 전체를 무한정 보내지 않고, 필요한 부분만 요약하거나 도구를 활용해서 토큰 낭비를 줄이기도 함.

## 7. MCP란?

`MCP(Model Context Protocol)`는 AI가 외부 도구나 시스템과 더 잘 연결될 수 있게 도와주는 방식 또는 규약

쉽게 말하면, AI가 단순히 말만 생성하는 걸 넘어서 아래 같은 작업을 할 수 있게 연결해주는 개념

- 파일 읽기
- 데이터 조회
- 외부 API 호출
- 특정 도구 실행
- 개발 환경과 상호작용

### 왜 MCP가 중요한가

AI는 기본적으로 텍스트 생성에는 강하지만, 실제 작업을 수행하려면 외부 도구와 연결

예를 들면:

- "현재 프로젝트 파일을 읽어줘"
- "에러 로그를 분석해줘"
- "데이터베이스에서 값을 조회해줘"

이런 요청은 모델 혼자서는 할 수 없고, 도구 연결이 필요함.  
이때 MCP 같은 구조가 있으면 AI가 필요한 문맥과 도구를 더 체계적으로 활용할 수 있음.

## 8. 핵심 요약

- AI 모델은 기본적으로 무상태라서 기억은 외부 시스템이 관리
- 대화는 보통 `messages` 배열과 `role`, `content` 구조로 전달
- `system`, `user`, `assistant` 역할을 통해 대화 흐름이 구성
- 프롬프트를 잘 작성할수록 원하는 결과를 더 안정적으로 얻을 수 있음
- 토큰은 비용, 성능, 문맥 길이와 직접 연결
- MCP는 AI가 외부 도구와 연결되어 실제 작업을 수행하도록 돕는 개념

## 9. 한 줄 정리

AI는 스스로 모든 걸 기억하고 실행하는 존재라기보다,  
`좋은 프롬프트 + 저장된 대화 기록 + 외부 도구 연결`을 통해 똑똑하게 동작하는 시스템이라고 이해하면 쉬움.
