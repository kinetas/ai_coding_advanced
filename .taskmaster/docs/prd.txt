# 개요
[제품의 개략적인 개요를 여기에 작성하세요. 어떤 문제를 해결하는지, 누구를 위한 것인지, 왜 가치 있는지 설명하세요.]

**어떤 문제를 해결하는지**
이동 중이나 바쁜 일상에서 관심 분야(예: 농산물 시세)의 최신 정보를 빠르게 확인하기 어렵고, 데이터를 직접 분석하려면 전문 도구나 복잡한 절차가 필요함.

**누구를 위한 것인지**
- 농산물 거래자(도매·소매업자, 농민): 시세·예측 정보가 필요한 사람
- 특정 분야 최신 정보를 찾는 일반 사용자
- 키워드로 맞춤 분석을 하고 싶은 데이터 활용자

**왜 가치 있는지**
3~4번의 터치만으로 카테고리→분야→중분류→세부내용을 선택해 관련 정보 분석을 바로 볼 수 있어, 휴대폰으로 이동 중에도 쉽게 활용 가능함.

# 핵심 기능
[제품의 주요 기능을 나열하고 설명하세요. 각 기능마다 다음을 포함하세요: 무엇을 하는지, 왜 중요한지, 대략적으로 어떻게 동작하는지]

1. **분석 제공**
   - 무엇을 하는지: GPS API로 내 위치를 기준으로 관심 분야를 선택하면, 평균 가격·가격 상승/하락 예측 등 서버에서 미리 계산해 둔 통계·분석 정보를 제공함.
   - 왜 중요한지: 사용자가 별도 계산 없이 즉시 의사결정에 필요한 인사이트를 얻을 수 있음.
   - 어떻게 동작하는지: 사용자가 위치 허용 후 분야 선택 → 서버가 해당 지역·분야 분석 결과 조회 → 화면에 표시.

2. **정보 수집(ETL 자동화)**
   - 무엇을 하는지: 크롤링 등으로 정보를 수집하고, ETL 파이프라인을 통해 자동으로 정제·최신화함.
   - 왜 중요한지: 수동으로 데이터를 갱신하지 않아도 최신 정보를 유지할 수 있음.
   - 어떻게 동작하는지: 스케줄러가 크롤러 실행 → 수집 데이터 변환·적재 → DB에 저장.

3. **워드클라우드**
   - 무엇을 하는지: 수집된 정보를 바탕으로 분야별로 자주 노출되는 단어를 워드클라우드로 시각화함.
   - 왜 중요한지: 한눈에 핵심 키워드와 트렌드를 파악할 수 있음.
   - 어떻게 동작하는지: 분야별 텍스트 추출 → 토큰화·빈도 집계 → 워드클라우드 생성 → DB 저장 후 프론트에서 표시.

4. **키워드 맞춤 분석 및 SNS 공유**
   - 무엇을 하는지: 사용자가 원하는 키워드(예: ~의 ~에 ~가 ~하는가)를 입력하면, 지역별 합계·평균·예측·분산 등을 분석하고, SNS API로 공유할 수 있게 함.
   - 왜 중요한지: 사용자 맞춤 분석과 결과 공유로 활용도와 확산이 높아짐.
   - 어떻게 동작하는지: 사용자 키워드 입력 → 서버가 DB에서 조회·분석 → 결과 반환 → (선택) SNS 공유 API 호출.
]

# 사용자 경험
[사용자 여정과 경험을 설명하세요. 다음을 포함하세요:
- 사용자 페르소나
1. 농산물 거래자: 도매·소매업자, 농민 — 시세·예측 정보로 거래 타이밍 결정
2. 관심 분야 탐색자: 특정 분야 최신 정보를 찾는 일반 사용자 — 이동 중에도 빠르게 정보 확인
3. 데이터 분석자: 키워드로 직접 분석하는 사용자 — 지역별·기간별 맞춤 분석
4. 정보 공유자: 분석 결과를 SNS로 공유하는 사용자 — 결과를 쉽게 공유하고 싶음
- 주요 사용자 플로우
GPS위치확인 -> 카테고리 선택 -> 중분류 선택 -> 세부 내용 선택 -> 관련 정보 분석 표시
분야 선택 -> 해당 분야의 주요 키워드 워드클라우드 확인
키워드 입력 -> 분석 실행 -> 결과 확인 -> SNS로 공유
- UI/UX 고려사항
서버에서 제공해주는 분석 정보나 워드클라우드는 휴대폰으로도 쉽게 볼 수 있도록 반응형 ui 필요
]
